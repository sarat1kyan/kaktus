Enterprise Security Features

Multi-Distro Support

Compatible with RHEL 7/8/9, CentOS, Debian 10/11/12, and Ubuntu LTS distributions. Automatically detects your system and applies appropriate hardening measures.

Comprehensive Modules

Eight specialized security modules covering user security, SSH hardening, kernel parameters, file permissions, firewall configuration, services management, auditd, and MAC policies.

Safety-First Design

Built-in dry-run mode, comprehensive impact analysis, automated configuration backups, and one-click rollback capabilities ensure your system stays operational.

Compliance Ready

Implements industry-standard security frameworks including CIS Benchmarks, DISA STIG, NIST 800-53/171, and OWASP guidelines for comprehensive compliance.

Automation Friendly

Support for JSON/YAML configuration profiles, non-interactive execution mode, and comprehensive logging make it perfect for CI/CD pipeline integration.

Detailed Reporting

Generate comprehensive security audit reports in multiple formats with color-coded severity indicators, compliance mapping, and actionable recommendations.

5-Minute Quick Start

user@server:~
# 1. Install Kaktus
$ git clone https://github.com/sarat1kyan/kaktus.git
$ cd kaktus
$ sudo ./install.sh
# 2. Run initial audit
$ sudo kaktus --audit-only
# 3. Preview changes (dry-run)
$ sudo kaktus --dry-run
# 4. Apply hardening
$ sudo kaktus

Pre-flight Checklist

🔒 Backup Strategy

Always have a full system backup before hardening production systems.

🖥️ Console Access

Ensure out-of-band access (IPMI, iLO, DRAC) - not just SSH.

🧪 Test Environment

Validate hardening on non-production systems first.

📋 Service Inventory

Document which services and daemons your system requires.

Security Modules

Module Key Features Impact
User Security UID 0 checks, umask hardening, password aging, disable system accounts High
SSH Hardening Disable root login, key-only auth, strong ciphers, connection timeouts High
Kernel (sysctl) TCP/IP hardening, ASLR enablement, ICMP redirect protection Medium
File Permissions Fix critical file permissions, remove world-writable files High
Firewall Auto-detect & configure iptables/nftables/firewalld/ufw Critical
Services Disable unnecessary services, remove development tools Medium
Auditd Install & configure comprehensive audit logging rules High
SELinux/AppArmor Enforce mandatory access controls, manage security policies Critical

Usage Examples

Common Use Cases

# Production Web Server
sudo kaktus --modules ssh,firewall,kernel,services
# Database Server
sudo kaktus --modules user_security,ssh,kernel,file_permissions,auditd
# Development Environment
sudo kaktus --config /etc/kaktus/profiles/development.yaml
# Compliance Audit
sudo kaktus --audit-only --report compliance_audit.json

Quick Commands

# Preview what would change
sudo kaktus --dry-run | grep "Would"
# Harden only SSH & firewall
sudo kaktus --modules ssh,firewall
# Non-interactive mode for automation
sudo kaktus --non-interactive --config production.yaml
# Roll back to previous state
sudo kaktus --rollback 20240115_143022_pre_hardening
